# Садиба «Чугайстер»

Навчальний вебпроєкт для представлення садиби в Карпатах та прийому заявок на бронювання.

## Можливості

- перегляд головної сторінки з описом садиби та перевагами відпочинку;
- перегляд варіантів проживання з цінами та додатковими послугами;
- онлайн-форма бронювання з валідацією полів;
- автоматичний попередній розрахунок вартості;
- збереження заявок у локальній базі даних Microsoft Access;
- надсилання email-повідомлень через EmailJS;
- локальний Node.js сервер для перегляду сторінок і роботи API.

## Стек технологій

- `HTML5`
- `CSS3`
- `JavaScript`
- `jQuery`
- `Node.js`
- `EmailJS`
- `Microsoft Access`

## Структура проєкту

- `index.html` - головна сторінка.
- `rooms.html` - сторінка з описом будиночків.
- `booking.html` - сторінка онлайн-бронювання.
- `style.css` - стилі інтерфейсу та адаптивність.
- `script.js` - клієнтська логіка взаємодії та валідації.
- `local-preview-server.js` - локальний HTTP-сервер і API.
- `booking-database.js` - вузол інтеграції між Node.js та Access.
- `access-db.ps1` - ініціалізація схеми БД та операції з бронюваннями.
- `images/` - ілюстрації для сторінки проживання.
- `start-site.bat` - швидкий запуск локального сервера і браузера.

## Запуск

1. Відкрити папку `проєкт1`.
2. Запустити файл `start-site.bat`.
3. Дочекатися старту локального сервера.
4. Відкрити у браузері сторінку `http://localhost:8016/booking.html`.

## API

- `GET /api/health` - перевірка доступності сервера.
- `GET /api/bookings` - список останніх бронювань.
- `POST /api/bookings` - створення нового бронювання.

## База даних

У проєкті використовується файл `проєкт1.accdb`. Під час першого запуску скрипт `access-db.ps1` перевіряє існування таблиць `Клієнти` та `Бронювання`, а за потреби створює або оновлює їх структуру.
